CVE-2025-20260

Security Research · CVSS 9.8 Critical · Cisco ClamAV

Published the first public proof-of-concept for CVE-2025-20260 - a CVSS 9.8 heap-buffer-overflow in the PDF scanning path of ClamAV (used inside Cisco Secure Email/Web). When PDF scanning and the max-file-size limit are configured a certain way, an oversized object lets attacker-controlled data write past an allocation, corrupting heap metadata; the practical impact is a clean bypass of malware scanning (and a DoS path) - ATT&CK T1562, Impair Defenses. The repo walks through the vulnerable code, the trigger conditions, a minimal malicious-PDF generator, and the affected/fixed version matrix, plus a full CVSS 9.8 write-up.

Glitchy

Hardware Security · Open Source

A no-code GUI front-end for voltage- and clock-glitch fault injection against microcontrollers. Lets a researcher define a glitch campaign visually - target trigger, offset/width sweep ranges, repeat counts, success oracle - then runs the parameter search, logs every attempt, and replays the winning glitch. Padding Oracle Attack on AES-CBC

CMU · Applied Cryptography · Jan-Feb 2025

A from-scratch implementation of the classic PKCS#7 padding-oracle attack against AES-128-CBC. Given only an oracle that leaks whether a ciphertext decrypts to valid padding, the tool recovers plaintext byte-by-byte (manipulating the preceding block to walk padding values 0x01…0x10) and then runs the encryption direction - chaining forged blocks to produce a ciphertext that decrypts to an arbitrary attacker-chosen message without ever knowing the key. Includes a deliberately vulnerable server, the attack client, and notes on why authenticated encryption (AES-GCM / Encrypt-then-MAC) defeats it.

Structured security evaluation of the DJI Mavic Mini (WM160). Reverse engineered the DJI DUML application protocol over Enhanced Wi-Fi (UDP) from monitor-mode captures across the full flight state machine - found AES-256 per-session encryption that resisted passive decryption, but no per-message replay protection on non-safety-critical movement commands (demonstrated controlled replay on team-owned hardware). Findings are in responsible disclosure with the advisor.

X.509 Certificate Chain Validator

CMU · Network Security

A from-scratch Python implementation of X.509 certificate-chain validation as used in TLS - building the path from leaf to a trusted root, checking signatures with the issuer's public key, enforcing validity windows, Basic Constraints / pathLenConstraint, Key Usage & Extended Key Usage, and name matching (SAN, wildcards).
